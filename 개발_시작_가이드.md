# 🚀 개발 시작 가이드

## ✅ 1. 지금 여기에서 개발할 수 있습니다!

Google AI Studio의 모든 코드가 현재 프로젝트에 통합되었으므로, **지금부터 Cursor에서 개발**할 수 있습니다!

---

## 📋 개발 환경 설정

### 1단계: 의존성 설치 (처음 한 번만)

프로젝트 폴더에서 터미널을 열고 실행:

```bash
npm install
```

또는 배치 파일 사용 (Windows):
- `start-dev.bat` 파일 더블클릭 (자동으로 설치 후 실행)

---

## 🎯 개발 서버 실행

### 방법 1: 배치 파일 사용 (가장 간단 ⭐)

**Windows 사용자:**
1. `start-dev.bat` 파일 더블클릭
2. 브라우저가 자동으로 열림 (보통 `http://localhost:5173`)

### 방법 2: 터미널에서 실행

**PowerShell 또는 CMD에서:**

```bash
# 프로젝트 폴더로 이동
cd C:\Users\USer\Documents\accounting-ledger-doozon

# 개발 서버 실행
npm run dev
```

**실행 후:**
- 브라우저가 자동으로 열리거나
- 터미널에 나타나는 주소를 브라우저에 입력 (예: `http://localhost:5173`)

---

## ❓ 2. 실행파일이 필요한가요?

### 답: **아니요, 실행파일이 필요 없습니다!**

이 프로젝트는 **웹 애플리케이션**이므로:

✅ **개발 시**: 브라우저에서 실행 (`npm run dev`)
- 코드 수정 시 자동으로 새로고침 (Hot Reload)
- 실시간으로 변경사항 확인 가능

✅ **배포 시**: 웹 서버에 배포
- 빌드 파일(`dist` 폴더)을 웹 서버에 업로드
- 예: Vercel, Netlify, GitHub Pages 등

---

## 📦 빌드 (배포용)

배포하려면 빌드 파일을 생성합니다:

```bash
# 프로덕션 빌드
npm run build

# 빌드된 파일 미리보기
npm run preview
```

빌드 후 `dist` 폴더가 생성되며, 이 폴더의 내용을 웹 서버에 업로드하면 됩니다.

```bash

# dist 폴더를 웹 서버에 업로드
```

## 🔧 문제 해결

### 포트가 이미 사용 중인 경우

다른 포트로 실행:
```bash
npm run dev -- --port 3000
```

### 의존성 오류

다시 설치:
```bash
rm -rf node_modules package-lock.json
npm install
```

### 캐시 문제

Vite 캐시 삭제 후 재실행:
```bash
rm -rf node_modules/.vite
npm run dev
```
