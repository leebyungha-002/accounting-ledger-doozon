# 🛡️ 오류 방지 가이드

오늘 발생했던 오류들을 다시는 겪지 않기 위한 체크리스트와 개선 방안입니다.

## 📋 오늘 발생한 주요 오류들

1. **JSX 구문 오류** - 태그 닫기 누락, 중첩 구조 오류
2. **Vite 플러그인 오류** - `@vitejs/plugin-react` 패키지 누락
3. **React Ref 오류** - `counterDrilldownRef` 선언 누락
4. **상태 관리 오류** - `setState` 호출 누락 또는 잘못된 값

## ✅ 코드 작성 전 체크리스트

### 1. JSX 구조 확인
- [ ] 모든 열린 태그(`<div>`, `<Card>`, `<ScrollArea>` 등)가 닫혔는지 확인
- [ ] 조건부 렌더링(`{condition && <Component />}`)의 닫는 괄호 `)}` 확인
- [ ] 중첩된 컴포넌트의 들여쓰기 일관성 확인

### 2. React Hooks 확인
- [ ] `useRef`로 선언한 ref가 실제로 사용되는지 확인
- [ ] `useState`로 선언한 state가 올바르게 사용되는지 확인
- [ ] `useEffect`의 의존성 배열이 올바른지 확인

### 3. Import 확인
- [ ] 사용하는 함수/컴포넌트가 모두 import 되었는지 확인
- [ ] 사용하지 않는 import는 제거

### 4. 타입 확인
- [ ] TypeScript 타입 오류가 없는지 확인
- [ ] `any` 타입 사용을 최소화

## 🚨 오류 발생 시 대응 절차

### 1단계: 터미널 오류 확인
```
- 빌드 오류 메시지 확인
- 라인 번호 확인
- 오류 타입 확인 (Syntax Error, Type Error 등)
```

### 2단계: 브라우저 콘솔 확인
```
- F12로 개발자 도구 열기
- Console 탭에서 에러 메시지 확인
- Network 탭에서 500 오류 확인
```

### 3단계: 파일 확인
```
- 오류가 발생한 파일 열기
- 해당 라인 주변 코드 확인
- 최근 수정한 부분 확인
```

### 4단계: 캐시 클리어 (필요시)
```
- 서버_완전_재시작.bat 실행
- 또는 수동으로:
  1. 서버 중지
  2. .vite 폴더 삭제
  3. node_modules/.vite 삭제
  4. 서버 재시작
```

## 📝 코드 작성 모범 사례

### JSX 구조 작성 시
```tsx
// ✅ 좋은 예: 명확한 구조와 들여쓰기
{activeCard === 'general' && (
  <div className="space-y-4">
    <Card>
      <CardContent>
        {/* 내용 */}
      </CardContent>
    </Card>
  </div>
)}

// ❌ 나쁜 예: 닫는 태그 누락
{activeCard === 'general' && (
  <div className="space-y-4">
    <Card>
      <CardContent>
        {/* 내용 */}
      </CardContent>
    </Card>
  // </div> 누락!
)}
```

### React Hooks 사용 시
```tsx
// ✅ 좋은 예: ref 선언과 사용
const myRef = useRef<HTMLDivElement>(null);

useEffect(() => {
  if (myRef.current) {
    myRef.current.scrollIntoView();
  }
}, [dependency]);

// ❌ 나쁜 예: ref 선언 누락
// const myRef = useRef<HTMLDivElement>(null); // 누락!

```
